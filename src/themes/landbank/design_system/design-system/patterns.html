Compositions · Patterns

Primitives, composed.

The recurring multi-primitive compositions of the ACLB system — data states, the dashboard kit, record cards, forms, and admin chrome. Each pattern here appears on at least one live page or admin surface; each is rendered live below, on a specimen canvas.

01 · Data states

Every data section owns three non-data states. They render inside the section's card, at the section's size — the page never collapses or reflows while data settles.

Empty state
used on: properties · datasets
No properties match these filters

Try removing a filter, or browse the full ledger.

The empty state names the cause (the filters) and offers the way out — never a bare "no results."

Loading state
used on: every data section

Skeletons, no spinners — the skeleton keeps the section's shape

Error state
used on: every data section
This data couldn't load

Something went wrong on our end. The rest of the page is unaffected.

Copy stays cause-agnostic; rose is reserved for exactly this moment.

The pattern test

The simplest test: if a composition appears on three or more pages, it's a pattern and it belongs on this page. And the rule cuts both ways — anything not documented here or on components.html doesn't ship.

Layouts & LayoutGroups

The structural layer: every DMS page is wrapped in a Layout (page chrome + nav slots), then divided into LayoutGroups (bands of content). This page documents each named variant, its wrapper class strings, and when to use it.

Page Canvas

Every MNY page sits on a topographic-line texture (topolines.png) masked behind a translucent color wash. The outerWrapper applies this as a tiling 500px background. LayoutGroups then either sit directly on this canvas (header — transparent) or float above it on white cards (content — opaque).

How groups interact with the canvas: The header LayoutGroup is transparent — its title and eyebrow text sit directly on the topo texture. The content / default groups use bg-white rounded-[12px] shadow-md on wrapper2, creating white cards that float above the canvas. The darkSection group applies its own dark topo overlay via wrapper1, creating a full-bleed dark band.

Site

└─ childWrapper

LayoutGroup Wrapper Class Reference

Verbatim class strings for each named LayoutGroup variant. Product LayoutGroups use mr-auto on wrapper2, NOT mx-auto — only auth is centred.

Variant wrapper1 wrapper2 Alignment

auth w-full h-full flex-1 flex flex-row w-full h-full flex-1 flex flex-row min-h-screen centred (mx-auto on inner form)

mr-auto rule: The content and default variants let wrapper2 fill the full width instead of centering — sections within them are capped by the sectionArray's layouts.centered class. This keeps content flush with the SideNav's right edge rather than floating in the centre of wide viewports.

Section Width — centered vs. full-width

Inside any LayoutGroup, the sectionArray picks one of two width classes based on group.full_width.

group.full_widthClass appliedResult
'off' (default) layouts.centered → max-w-[1020px] mx-auto px-0 lg:px-[56px] Sections capped at 1020px, left-aligned
'show' layouts.fullwidth → "" (empty) Sections fill parent — for maps, wide tables
Pairing with LayoutGroup variant: For a true edge-to-edge map workbench, use darkSection or a custom variant that drops max-w from wrapper2, and set group.full_width: 'show'. Setting fullwidth alone on a content group is still clipped by wrapper2's implicit width.

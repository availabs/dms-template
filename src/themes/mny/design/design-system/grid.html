3.0 Structure

Column Grid

The sectionArray grid that every Section sits on inside any LayoutGroup. This is the inner layout — not the page chrome (see Layouts). Every Section picks a span from this vocabulary; no freehand widths.

Grid Specification

PropertyValueNotes
gridSize1212-column grid; mobile collapses to 6 columns (grid-cols-6 md:grid-cols-12)
defaultSize2 (full-row span)New sections default to col-span-12
containerw-full grid grid-cols-6 md:grid-cols-12gap-0 — gutter is section padding
layouts.centeredmax-w-[1020px] mx-auto px-0 lg:px-[56px]cap for content sections
layouts.fullwidth"" (empty)sections fill parent LayoutGroup
sectionPaddingp-2 or p-3per-section gutter; border/bg rendered on inner box

gap-0 / padding-gutter / inner-box model: The section grid is gap-0. Gutter between sections is per-section padding (e.g. p-3). A visible card border / rounded corners / background render on an inner element inside that padding. This lets adjacent sections either sit apart (normal) or fuse visually (compound card: shared-edge padding zeroed + coordinated border corners).

Compound Card Pattern

Two adjacent sections fused into one visual card (e.g. a stat header above a chart). Achieved by zeroing the shared-edge padding and coordinating border corners — not a single section.

Compound card = two DMS sections: top section padding p-3 pb-0 + inner rounded-t-[12px] border-b-0; bottom section padding p-3 pt-0 + inner rounded-b-[12px] border-t-0. Maintains 1:1 mapping with DMS sections — no single-section hacks.

Authoring Rules

  • →Default to full-row (size: 2). Almost every section starts as full-width; narrow as needed.
  • →Favour symmetric splits (1/2+1/2, 1/3+1/3+1/3, 1/4×4) over asymmetric splits unless the data justifies asymmetry.
  • →mobile behaviour: all sizes except 2 collapse to col-span-6 (half-mobile) — verify stacking order makes sense at 375px.
  • →Map / workbench sections should use group.full_width: 'show' on the wrapping LayoutGroup and size: 2 on the section.
  • →Compound cards must be two separate sections with zeroed shared-edge padding — never one section with multiple components hard-coded in.

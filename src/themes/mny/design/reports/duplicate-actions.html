The verdict

One in ten action rows
is a second copy of another.

An exact duplicate here means two or more rows that carry the same action name and the same description in the same municipality. That is not a design choice or a near-match — it is the same action, in the same place, recorded more than once. There are 1,945 such groups covering 1,960 redundant rows: remove the extras and 18,378 actions become 16,418, with nothing lost.

The duplication is not spread across the state. Ninety-six percent of it comes from five county annexes — New York City, Chautauqua, Monroe, Broome and Madison. In Monroe and Madison roughly half of all actions are duplicated; in the other 57 counties the rate is essentially zero. Duplication is a property of a handful of imports, not of the data-entry process.

The row ids of a duplicated pair sit far apart (a median of 5,962 ids between them), so this is not a stray copy-paste of an adjacent row. But two in five pairs share the exact same creation date. The signature is unmistakable: each of these five annexes was loaded into the dataset twice — two overlapping source workbooks, or a re-run of an import — leaving a full second copy with fresh ids.

Because every duplicate is by definition the same action in the same place, deleting the extras removes no distinct action. Thirty-nine percent of groups are identical (or differ only where one copy left a field blank) and collapse mechanically. The rest differ only in secondary attributes — a project-timeline estimate, a status, a hazard tag — and are resolved by a simple rule: keep the most complete, most recently updated copy.

Three signals, read together, point to one mechanism: a duplicated bulk import of specific county annexes.

Is It Safe To Delete?

Every group is the same action in the same place, so no distinct action is ever lost. The only question is which copy’s secondary values to keep. Sorted by how much judgement that takes:

Identical — or blank-vs-filled

Copies match on every substantive field, or differ only where one left a field empty.

Auto
767groups · 39%

Differ in a secondary field

Same action; copies disagree on an attribute like project timeline or status. Keep the fuller / newer copy.

Rule
1,178groups · 61%

The fields that disagree are almost never the action itself. The most common are the project-timeline estimate (834 groups), action_status and implementation_status (455 / 428), and hazard tags (419 / 376). None of those change which action it is — only how one copy was annotated. And 31 of the redundant rows already carry isValid = false, the dataset’s own “this row is bad” flag.

Recommended path

De-duplicate with a keep-the-richer-copy rule

1 · Group

Match on normalized name + description + locality. That key is conservative: it only flags rows that are the same action in the same place.

2 · Keep one

Within each group keep the survivor that is valid, most complete, and most recently updated; drop the rest. 39% need no choice at all.

3 · Start with the five

NYC, Chautauqua, Monroe, Broome and Madison return 96% of the rows. Clearing those five annexes fixes almost the entire problem and every action total that double-counts them.

Net effect: 18,378 → 16,418 action rows (−10.7%), no distinct action removed, every county and statewide count corrected for double-counting, and fewer actions stacked on the same map point. The duplicate-heavy counties are, notably, not the template-heavy ones — see the Boilerplate Actions report for the complementary pattern.

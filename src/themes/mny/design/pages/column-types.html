COLUMN TYPES — DESIGN DRAFT

Six new column types and format functions for site management spreadsheets. Pages get page_publish and page_link. Sections get component_type, source_id, view_id, and element_preview.

COLUMN TYPE LEGEND
P
PAGE_PUBLISH
Status pill (Draft / Changes / Published) with inline Publish and Discard buttons. Fetches the full page via Falcor before calling editFunctions.publish. Pages only.
type: 'page_publish' · column key: 'published'
L
PAGE_LINK (formatFn)
Applies to any text column. Renders the value as a <Link> using row.url_slug. Set Format → page_link on the Title column.
formatFn: 'page_link' · reads row.url_slug for href
C
COMPONENT_TYPE (virtual)
Extracted from element['element-type'] via SQL expression key. Works on all existing data without migration. Filterable. Sections only.
key: "data->'element'->>'element-type' as component_type"
S
SOURCE_ID / VIEW_ID (virtual)
Extracted from element.externalSource nested JSON. Filter sections by data source without any re-save. Sections only.
key: "data->'element'->'externalSource'->>'source_id' as source_id"
E
ELEMENT_PREVIEW
A Preview button column. On click: fetches the section's full data blob via Falcor, looks up the component from the registry, and renders it inside a modal using SectionComponents.ViewComp. Sections only. No edit capability — preview only.
type: 'element_preview' · opens modal · fetches byId → data → element

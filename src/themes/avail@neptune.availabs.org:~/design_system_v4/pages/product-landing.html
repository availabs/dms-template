name

tessera — a data platform where pages, tables, charts, and maps are four ways of rendering the same row.

synopsis

$ tessera render <row> --as page|table|chart|map

description

There is exactly one datatype — a typed row. Sites, themes, patterns, pages, sections, datasets, views, columns, and joins are all that same row, distinguished only by their type string. A join is two rows pointing at each other. A theme is a row whose data is style. That is why a site builder, a data platform, and a mapping tool can be one product without seams — there is nothing to seam between.

One representation

Many renderings.

A page is a row. A chart is a row. A map is a row. There is nothing else to learn.

--as page

WYSIWYG sections, draft/publish, per-page permissions, deep theming.

--as table

Typed, filterable, joinable columns over internal or external Postgres.

--as chart

Bar, line, scatter, pivots, computed columns — URL-synced filters throughout.

--as map

MapLibre sections with symbology, PMTiles, geo-aware columns.

Layouts.

The structural layer: every page is a Layout (chrome: optional TopNav, optional SideNav, a content area) stacking LayoutGroups (bands) that host Sections. Nothing on any Tessera page steps outside this nest. Toggle annotate in the nav to see this page confess its own structure.

The nest — Site → Layout → LayoutGroup → Section → Component

layoutgroup · a band — header / content / footer

The mosaic reading: the Layout is the wall, a LayoutGroup is a course, a Section is a tessera, the p-3 gutter is the grout. The composition — not the tile — is the design.

LayoutGroup variants

the bands — every one the brand ships

Tessera bands are unboxed courses of stone: no shadows, no rounded shells — a band changes stone (basalt ↔ slab) and rules a mortar line where it meets its neighbor. Bands keep y-padding minimal (py-4); sections own their spacing through the p-3 grout.

hero

hero

Basalt + the cell grid ornament, mortar rule below. Marketing lead bands; hosts the landing page's three.js section.

header

header

Plain basalt, mortar rule below. Page titles, breadcrumbs, the metadata strip on product pages.

content · the default

Basalt ground; sections read as slab tiles set into it. Most bands on most pages.

content_slab

The tint band: slab ground, mortar rules top and bottom. Alternates with content to pace long pages; inner tiles flip to basalt.

login:

auth

Cell-grid ground, one centred section (max-w-md, mx-auto — the intentional centring). Sign-in is a login prompt.

footer

Slab, mortar rule above, quiet link columns, the colophon and one tessera course. Ends every page.

workbench

Full-bleed, no cap (px-0). Map canvases, wide spreadsheets, the page editor. Pair with full_width: show (see [04]).

Rule: product bands (Layout app) use mr-auto, never mx-auto — the band hugs the SideNav; the asymmetric pl-12 pr-8 gives breathing room off the nav's edge, and mr-auto consumes the excess viewport on the right. The codebase default (max-w-[1020px] mx-auto) drifts content away from the sidebar on wide monitors — don't use it. Documented exception: on Layout default (marketing — no SideNav to hug) the same variants swap mr-auto … pl-12 pr-8 for mx-auto … px-6. auth centres by intent, everywhere.

[04]

Section width — regular vs full-width groups

Inside any band, the sectionArray picks one of two layout classes, keyed by the author-facing Full Width: off / show toggle:

className={`
  ${theme.container}
  ${theme.layouts[group.full_width === 'show'
      ? 'fullwidth' : 'centered']}
`}
group.full_widthclass applied
'off' (default)layouts.centered = 'max-w-[1280px] mr-auto'
'show'layouts.fullwidth = ''

Pair the toggle with the band. full_width: show inside a content band still caps at 1280px — wrapper2 itself carries the cap. A true edge-to-edge canvas needs the workbench variant (wrapper2 drops the cap) and full_width: show, together.

content + off — capped, left

content + show — still capped by wrapper2

workbench + show — edge to edge

Use workbench for maps, wide spreadsheets, and the page editor — not for ordinary content. The 1280 cap is correct for nearly every band.

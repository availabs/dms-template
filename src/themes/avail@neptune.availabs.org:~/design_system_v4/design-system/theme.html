Tokens.

Tessera is a terminal run on stone. The character cell — the smallest unit a console can set — is a tessera: a small square block placed in courses on a grid until a larger surface composes. This page is the brand's vocabulary before it is composed into anything: thirteen mineral colors, fourteen type tokens in four roles, one icon set cut on a 24px grid, a 4px space scale, a radius of zero, and the motion and ornament rules that keep the surface quiet.

[01]

Color

stone · ink · phosphor

Surfaces are dark stone; ink is chalk; the accents are the six phosphors — the ANSI palette named as minerals. Ground holds ≥ 85% of any surface. Phosphor green is the interaction color: prompts, focus, primary actions, success. The other phosphors appear only in the role listed on their chip. Tessera is dark-only by decision — the screen is stone, and stone is dark. There is no light mode to drift out of sync.

[02]

Type

14 tokens · 4 roles · two families

One terminal face carries three roles: JetBrains Mono is display (bold, raised), prose (regular, running output), and meta (medium, uppercase, tracked — labels and chrome). The fourth role is the stone's own voice: Cinzel, an incised Roman capital, reserved for inscriptions — the wordmark, band inscriptions, the colophon. Every text string on every page of this deliverable resolves to one of these fourteen tokens; color, italic, tabular-nums, and call-site uppercase are modifier axes, not new tokens.

[03]

Icons

63 glyphs · cut, not drawn

A custom set on a 24px grid: 1.75px stroke, square caps, mitre joins, no rounded corners. Wherever another set would use a dot, this one sets a small filled square — the cell is the atom. Registry source: theme/icons.js; names are the API (theme.Icons[name]). Every svg in this deliverable is tagged <!-- icon: Name --> or <!-- decorative -->.

Space · radius · elevation

Space — 4px base

12px (p-3) is load-bearing: it is the section gutter — the grout width. Component padding favors 16/20/24.

Radius — one value

0px, everywhere. A character cell has no corners to round. Affordance comes from inverse video and the focus frame, never from softness.

Elevation — three stops
flat
set · slab + mortar
lit · focus/hero only

No drop shadows. Depth is which stone you're standing on, not how far the card floats. The phosphor halo (shadow-glow) marks the focused or featured surface — one per viewport.

Motion

set, don't slide
momentspec
state flip (hover, tab, toggle)90ms · steps(1) or cubic-bezier(.2,0,.1,1)
reveal (section, menu, modal)140ms · opacity 0→1 + translateY(4px)→0 — a tile set down
cursor blink1.1s · steps(1) — the only loop the brand permits
hero assembly (three.js)courses settle row by row, then stillness; runs once
prefers-reduced-motionblink stops; reveals render placed; hero renders composed

Banned: parallax, marquees, looping backgrounds, eases longer than 200ms, anything that moves while the reader is still.

Ornament

five sanctioned marks · one per band, at most

The surface is ~90% plain stone. Ornament is punctuation, drawn only from this set — anything else is graffiti.

1 · The cell grid

.tes-cellgrid — the character matrix as faint mortar lines. Hero and tinted bands only; never behind running prose.

ready

2 · The block cursor

.tes-cursor — the signature tessera. Ends the hero headline, marks live/waiting states. One per surface.

tty1 · tessera80×24

$ _

3 · The tty frame

A labelled titlebar on a mortar frame — the brand's "terminal window" chrome for demos and code surfaces.

4 · The tessera course

One row of set cells, at most one lit — band punctuation for the hero and the footer. Never tiled into a field.

5 · The incised rule

.tes-rule — a groove cut into the stone (dark line over light). Long-form dividers; pairs with the incise voice.

[07]

Voice

man-page register
Principles
  • Specific over evocative. "A typed row in data_items" beats "a flexible building block."
  • Lowercase where the terminal would be. Prompts, statuses, and system strings are lowercase; sentences are sentences.
  • Exit codes, not apologies. Errors say what failed, where, and what to try. No "oops."
  • No exclamation marks, no superlatives. The system is right because it's right.
  • The verbs of authoring: compose, place, set, render, publish.
  • Earned metaphor. Tessera/mosaic language appears only when it names a real property of the system.

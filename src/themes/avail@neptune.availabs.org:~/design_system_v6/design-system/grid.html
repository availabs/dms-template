Grid — the sheet

Every section on every page sits on the same 12-column sectionArray grid. The graph-paper texture behind this text isn't decoration — it's that grid made visible, one 24px cell at a time. This page is the vocabulary: the spec, the spans, and the grout model that holds it together.

squint at this band's background — everything snaps to it. that's the whole system

01 · the spec

One grid for the whole brand

Twelve columns, no grid gap, 1200px cap. The same grid inside every LayoutGroup on every page — authors learn it once and it never changes underneath them.

setting value why
gridSize 12 fine-grained spans; matches how everyone already thinks
defaultSize "12" new sections land full-row; authors shrink deliberately
container w-full grid grid-cols-12 gap-0 gap-0 — the gutter is per-section padding, not a grid gap (§04)
layouts.centered max-w-content mr-auto 1200px cap, hugging the sidenav — never mx-auto on product pages
layouts.fullwidth '' sections fill the parent when the author flips Full Width: show
sectionPadding p-3 12px — exactly half a 24px sheet cell; two gutters meet at one cell
paddings { 0, 2, 3, 4, 6, 8 } the curated steps authors can pick per section; 0 fuses neighbours (§04)
gridviewItem dashed cobalt outline · cobaltSoft fill what a section looks like in the editor's "show grid" overlay (§05)

04 · the grout model

Gutters are padding, chrome is inner

Three rules make the whole system work: the section grid is gap-0; the gutter is each section's own p-3 padding; and the border / radius / background live on an inner box inside that padding. Two neighbours each contribute 12px of gutter — one full 24px sheet cell between cards. And because the gutter belongs to the section, zeroing it lets neighbours fuse into one card.

wrong — gap on the grid, chrome on the section

Looks the same today — but the gutter belongs to the grid, so no section can ever give it up. Fusing two sections into one card is now impossible, and this markup doesn't map to DMS.

right — gap-0, p-3 gutters, inner box

Each section owns its 12px of grout and the card chrome sits on the inner box. Set a section's padding to 0 on a shared edge and the two cards become one — see below.

Worked example: two sections, one card

The header and the chart below are separate sections in this band's grid — check the section labels on the right edge. They fuse because each zeroed the padding on the shared edge and split the border between them.

06 · picking spans

Picking a span

  • ·Default to 12. A full row is never wrong; a cramped half-row often is.
  • ·Reach for 6+6 or 4+4+4 before anything asymmetric — symmetric splits read as "peers", which is usually the truth.
  • ·Go asymmetric (8+4, 3+6+3) only when the data justifies it: a main thing with an aside, a reading column with rails.
  • ·Anything narrower than 3 isn't a section, it's a component inside one.
  • ·Below md everything stacks to 12 anyway — pick spans for the wide case, let the stack handle the rest.

Where to next

You've got the chrome (layouts) and the sheet (this page). What actually goes in the sections — buttons, cards, tables, forms — is components.html, and how they compose into recurring page moments is patterns.html.

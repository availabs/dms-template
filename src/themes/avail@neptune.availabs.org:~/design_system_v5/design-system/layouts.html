Layouts

The structural layer: which Layout a page wraps itself in, and how its bands of LayoutGroups stack. This page documents chrome shapes; the column grid the sections sit on has its own page, Grid.

The five-layer nest

Every Tessera surface — this page included — lives inside the same hierarchy. Nothing on any page in this system escapes it.

Site — routing & theme resolution

Layout — page chrome: TopNav · SideNav · content area

LayoutGroup — one band: hero / content / plate / ink / footer

Section — one editable block, spanning grid columns

Component — Card · Spreadsheet · Graph · Map · Lexical · …

Layout · default

Marketing & long-form pages. TopNav only; centered bands.

  • outerWrapper: min-h-screen flex flex-col bg-parchment
  • Bands center at cap 1200 with mx-auto px-6
  • Used by: the landing page, theme/layouts/grid docs

Layout · app

The workroom: authoring, admin, datasets. TopNav + SideNav.

  • SideNav: paper stock, w-60 border-r-2 border-ink
  • Bands hug the rail: mr-auto pl-12 pr-8 — never mx-auto
  • Used by: Components and Patterns docs

Layout · bare

Auth pages, embeds, print. No navs — one centered plate.

  • childWrapper fills the viewport; the auth group centers
  • The one place mx-auto is correct on app-side surfaces
  • See the sign-in pattern on Patterns

ink + sunburst

Sign-in/sign-up. A single max-w-md poster frame, centered both ways — the one intentional mx-auto band.

Wrapper class reference

The literal class strings, so alignment decisions can't drift. wrapper1 is the full-bleed band; wrapper2 is the capped inner container the sectionArray renders into.

The mx / mr rule. This marketing layout has no SideNav, so its bands center with mx-auto. On Layout app the platform rule applies: wrapper2 uses mr-auto with asymmetric pl-12 pr-8 so content hugs the SideNav instead of drifting toward the middle of wide monitors. Only auth keeps mx-auto on the app side.

Section width — regular vs full-width groups

Inside any LayoutGroup the sectionArray picks one of two layout classes, driven by the author-facing group.full_width toggle (Full Width: off / show in the group menu). Developers don't intervene.

group.full_width Class applied Result
'off' (default)layouts.centered — 'max-w-[1200px] mx-auto' (app: mr-auto)Sections capped at the brand width
'show'layouts.fullwidth — ''Sections fill the parent band

Pair it with the band. Setting 'show' inside a capped band still stops at the cap, because wrapper2 itself is capped. A true edge-to-edge canvas (a map workbench, a wide data table) needs a band whose wrapper2 drops its max-width and group.full_width: 'show'. v5 has no such band yet — when a Tessera workbench surface appears, add a workbench variant here rather than improvising one on a page.

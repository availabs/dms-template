Structure · the column grid

Grid — opus tessellatum

The regular bed of courses every section is set on. This page documents the sectionArray grid inside any LayoutGroup — the twelve columns a section spans, the grout between neighbours, the chrome on each tile. It is not the page chrome; that is Layouts.

The spec

One grid for the brand

Twelve columns, everywhere, on every LayoutGroup of every page. These are the canonical v3 values the theme translation reads; the grid itself never carries a gap.

The grout model

Sections are tesserae; the padding is the grout

The sectionArray grid is gap-0. The gutter between sections is per-section padding, and a section’s border, background, and lift render on an inner box inside that padding. This is what lets two sections sit apart or fuse into one visual card — and it is the most Tessera-true detail in the system.

One section, exploded
section (col-span-N) — the grid cell
padding = grout (p-3)
inner box (border + bg + lift)

Three layers, three duties: the cell spans columns, the padding keeps the course, the inner box wears the chrome. Content padding inside the card belongs to the component — it is not the gutter.

Two ways to cut the same course

Both rows below are three 4-column sections. Only one of them can an author ever fuse, tint, or zero.

Wrong — grid-cols-12 gap-6, margins between sections

The gap is the grid’s, not the section’s: no section can give it up, so no two sections can ever share an edge. The theme cannot zero it per side.

Right — grid-cols-12 gap-0, p-3 grout on each section

Same 24px grout to the eye — but the grout belongs to each section. Zero one side and two tiles fuse; that is the compound card below.

The compound card — two sections, one tile

A card header and its chart, authored as two stacked sections whose shared edge is zeroed and whose borders are coordinated. It maps 1:1 to two DMS sections an author can edit independently.

In the editor

What the author sees

Toggling “show grid” in the page editor draws the twelve courses as dashed grout outlines over bone (gridviewItem); the section being edited takes an oxide dashed outline (sectionEditing) with its span read out.

col 5–8 · span 4

The overlay is scaffolding, never shipped: it exists so an author placing a section sees the same courses this page documents.

Rules of placement

Picking a span

  • Default to the full row. Twelve is the resting span; anything narrower is a decision.
  • Prefer 6+6 and 4+4+4. Symmetric courses read as one floor.
  • Asymmetry must be earned by the data — 8+4 for a chart and its notes, never for taste.
  • Row-spans are punctuation, not rhythm. One hierarchy moment per surface.
  • If a section fights the grid, the section is wrong, not the grid.

Themes — visual identities for every site.

Each theme defines the color palette, typography, spacing, and component styles for one or more sites. Edit opens the component-level theme editor. Duplicate clones all token overrides.

DESIGN NOTES · GAPS VS. CURRENT

What this design adds over the current implementation.

Added
  • + Palette preview strip — 6-color swatch from actual theme tokens
  • + "Used by" list — shows which sites reference each theme
  • + Active/draft status badge — draft = not in use by any site
  • + Built-in vs. custom label in theme card
  • + Component coverage table — which components each theme defines
  • + Card layout instead of flat table — easier to compare visually
Implementation notes
  • → Color swatch: read from theme object's c.* palette (or first 6 button/background tokens)
  • → "Used by" requires cross-referencing patterns whose theme.selectedTheme matches
  • → Draft status: theme has no pattern using it
  • → Component coverage: inspect theme keys vs. DMS default theme keys

Surface & Canvas · the cards-on-pane rule

The page is a grey pane that floats white cards. Three z-layers: ink sidebar, grey content pane, white card. A fourth bone-tinted card is reserved for editorial / jurisdictional documents.

#eceef2

Good vs Bad
// GOOD
Section bg is pane (#ECEEF2). Content lives in a card with a hairline edge.
// BAD
Wall-to-wall white. No edge between section and content. Don't do this.

05

Icons · the registry

Icons are referenced by name, never by import. The names are the API: a theme entry like { caretIcon: "CaretDown" } resolves against this registry. Add a new icon to theme/icons.js and it's instantly available to every primitive that references it.

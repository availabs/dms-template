Card Layout

The Card section is the primary layout primitive in Tessera. It turns a data source into a grid of styled cells — no custom code required.

Core concept Admin UI v2.4+
Latest stable v2.4.1 View changelog →
Author empowerment first. Before writing a custom component, check if the Card's existing knobs can express what you need. The goal is to make the platform richer, not to build one-off solutions.

How Card sections work

A Card section binds to a data source — a dataset, a CMS type, or a computed view — and renders each row as a styled "card." The page author controls which columns appear, their visual style, their grid position, and a dozen other properties, all from the admin toolbar.

The key insight is that most layout decisions belong to the author, not the developer. Fonts, image sizes, column spans, grid density — these are configuration, not code.

The rendering pipeline

When a Card section loads, it runs three stages:

  1. Fetch rows from the bound data source (respecting auth filters)
  2. Apply formatFn transforms to each column value
  3. Render each column value using its registered column type's ViewComp

Grid configuration

Cards live inside a two-level grid. The cards grid controls how many card columns exist across the section. The cells grid controls the internal layout of columns within a single card.

JSON 
// Section config (stored in DMS, editable via toolbar)
{
  "cardsGridSize": 3,        // 3-column cards grid
  "cellsGridSize": 12,       // 12-col inner grid per card
  "columns": [
    {
      "col": "thumbnail",
      "type": "image",
      "cellSpan": 12,          // full width image
      "imageSize": "cover",
      "imageLocation": "top"
    },
    {
      "col": "title",
      "type": "text",
      "cellSpan": 12,
      "valueFontStyle": "displayMD"
    },
    {
      "col": "category",
      "type": "text",
      "cellSpan": 6,
      "valueFontStyle": "proseSM",
      "formatFn": "upper"
    }
  ]
}

Column types

Every cell in a card renders through a column type. Built-in types cover images, text, links, and dates. Themes can register additional types for domain-specific chrome (play buttons, status pills, etc.).

Type Description Key config
text Renders any value as styled text. Respects valueFontStyle and formatFn. valueFontStyle formatFn
image Renders an image URL as a responsive image cell. Controls aspect ratio and crop. imageSize imageLocation
link Wraps value as a navigable link. Can target internal DMS pages or external URLs. linkCol openExternal
date Parses an ISO date string and formats it using locale-aware format strings. dateFormat
theme-registered Custom types registered by themes for domain-specific chrome elements. theme Per-type cardHints

formatFn transforms

A formatFn is a pure function applied to a column value before it reaches the renderer. It's the right place for display-layer concerns: formatting numbers, uppercasing strings, combining two fields into one display value.

Don't reach for a custom component when a formatFn will do. If you need to display album · year from two fields, write a combining formatter — not a new column type.

Built-in formatFns

  • comma — Format number with comma separators: 1234567 → 1,234,567
  • abbreviate — Compact notation: 1200000 → 1.2M
  • date — ISO date → locale string
  • time — ISO datetime → HH:MM
  • upper — Uppercase transform
  • dollar — Prepend $ + comma format
  • percent — Multiply by 100, append %

Getting started: your first Card section

  1. 1
    Add a section to your page

    In the page editor, click + Add Section and choose Card from the section type list.

  2. 2
    Bind a data source

    Use the Source picker in the toolbar to connect the section to a dataset or CMS type. All available columns will appear in the column list.

  3. 3
    Configure columns

    Toggle columns on, set their spans, pick a type and font style for each. Changes preview live on the canvas.

  4. 4
    Set grid sizes

    Use the Cards grid slider to set how many columns of cards appear across the section. Use Cells grid for fine-grained internal column control.

  5. 5
    Publish

    Click Save in the top bar. The section is live immediately — no build step required.

The row format

One row. Every surface.

A page is a row. A section is a row. A dataset is a row. The same typed shape drives every surface in the product — and because the shape is shared, the parts compose without translation.

  • Six fields. Six. The whole vocabulary fits in a tweet.
  • Every plugin extends the same shape — no new schemas per integration.
  • One tessera export dumps your entire site as plain rows.
row.shape.ts 
// every row, everywhere, has the same shape
type Row = {
  app:        string;   // "the-drift"
  type:       string;   // "issues|page"
  parent:     string;   // "the-drift:site"
  data:       Record<string, unknown>;
  attributes: Record<string, unknown>;
  children?:  Row[];
};

// a page
{ app: "the-drift", type: "issues|page", data: { title: "Archive" } }

// a dataset row
{ app: "the-drift", type: "issues|14:data", data: { title: "Issue 14" } }

// a chart spec
{ app: "the-drift", type: "sub_growth|component", data: { kind: "line" } }

In use
CLI

Or skip the GUI entirely.

The CLI speaks the same row format. Init, edit, deploy. Push to hosted or to your metal with one flag.

~/projects/thedrift 
tessera init the-drift
  scaffolding site config… ok
  creating data sources… ok
  installing theme @tessera/editorial… ok
✓ initialized in 1.1s

tessera page create issue-archive --layout=default
  creating row { type: "issues|page", parent: "the-drift:site" }…
✓ page issue-archive created · open at http://localhost:4444

tessera deploy --host=cloud
  bundling 38 pages, 12 datasets… 1.2s
  uploading to thedrift.com… 480 KB
✓ deployed in 2.4s · view at https://thedrift.com

tessera tail thedrift.com
  → live · 2 publishes/min · 47 readers

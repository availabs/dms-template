v0.2 design system · column grid

Grid — every section sits on this column.

The column grid every Section is laid out on, inside any LayoutGroup. This is the sectionArray grid — 12 columns, defaultSize 12, max-width 1280px. It is not the page chrome (Layout / LayoutGroup, the bands that stack down a page) — that lives in layouts. Every Section in every example page snaps to one of the spans documented here; no freehand widths.

Read the layouts page → See components
scope → This page documents the column grid — the inner 12-column lattice Sections snap to inside every LayoutGroup. For the outer page chrome (Layout variants, LayoutGroup bands), see layouts.html.
The grid spec

12 columns, 1280px max, defaultSize 12.

Lingua v0.2 runs the modern 12-column sectionArray grid — finer-grained than the codebase default 6-column, matched to the contemporary web vocabulary. Every Section a user adds picks a size from the column-span vocabulary in this table; new sections that don't pick one default to a full row ("12"). The grid is the same across every LayoutGroup on every page in the system — one grid for the brand, no per-page reconfiguration.

Setting Value Note
gridSize 12 Number of columns the grid divides into. Codebase default is 6; Lingua follows the modern recommendation of 12 (≈8.3% per column).
defaultSize "12" What a new Section gets when no size is picked. Full row — authors must opt in to splitting.
layouts.centered max-w-[1280px] mx-auto Max content width when a group is not full-bleed. Matches --grid-max. Full-bleed groups override.
layouts.fullwidth "" Full-bleed groups (the dark code band, the footer) override centered to span the viewport.
sectionPadding p-3 · 12px Inner padding around each section's content. Modest in Lingua — the LayoutGroup's wrapper carries most padding.
container.gap gap-3 · 12px Gutter between columns. Matches --space-3. Tight enough to read as one composition; wide enough that adjacent sections breathe.
sizes 12 · 8 · 6 · 4 · 3 · aliases Column-span vocabulary. See §spans for examples and §fractional for the legacy 1/3/1/2/2/3/1 alias mapping.
rowspans 1 · 2 · 3 · … 8 Optional row-span override for sections that need vertical hierarchy. Default 1. See §rowspan.
gridviewGrid bg-zinc-50 h-full The "show grid" overlay backdrop authors see in edit mode. Subtle, off-white.
gridviewItem border-x border-dashed border-accent/30 The 12 column tracks visible behind sections when the editor toggles "show grid". Hairline accent-blue dashes.
where this comes from → Theme keys live under pages.sectionArray.styles[0]. The Lingua theme overlay (v0.3) populates these from the values above; the translation skill maps every row in this table to one key in theme.js. Fractional aliases

Legacy fractions, in 12-col terms.

The old codebase grid used fractional size keys — 1/3, 1/2, 2/3, 1. Lingua keeps these aliases so existing pages don't break: each one maps cleanly into the 12-column equivalent. New work should prefer the raw integer ("4", "6", etc.) since it reads more directly against the column ruler — but either is valid.

Legacy alias 12-col equivalent Common name
"1/3" "4" Third. Three-up grids, sidebar widgets.
"1/2" "6" Half. Side-by-side intro pairs, two-up grids.
"2/3" "8" Two-thirds. Main column with a 4-track rail.
"1" "12" Full row. The defaultSize.
how the theme resolves a size → The sectionArray theme's sizes map keys both forms to the same Tailwind col-span-N string. A Section with size: "1/2" resolves to the same class as one with size: "6". Authors can pick either; the rendered row is identical.
Row-span vocabulary

Vertical spans — for hierarchy moments only.

A Section can claim more than one row of the implicit grid via rowspan. Below: a rowspan: 1 default next to a rowspan: 2 tall and a rowspan: 3 very tall, all on the same 12-column grid. Row-spans are powerful and dangerous — they reorder how the grid auto-flows and can leave gaps. Reach for them only when the hero of a band genuinely deserves vertical weight.

rowspan 1 · span 4 Default height.
rowspan 2 · span 4 Two rows tall. Use for a primary chart or a featured Card amongst secondaries.
rowspan 3 · span 4 Three rows tall. The hero of the band — a code block, a screenshot, a key insight.
rowspan 1 · span 4 Sits beside the row-2 tile.
rowspan 1 · span 4 Sits beside the row-2 tile.
cost of a rowspan → Row-spans reorder how the grid packs. A rowspan-3 tile in column 1 forces everything else to flow around it; subsequent sections may snap to lower rows than the eye expects. Use a row-span when the visual hierarchy demands it — not as a default.
In-editor grid view

What authors see when "show grid" is toggled.

In the page editor, an author can toggle a "show grid" overlay that draws 12 dashed column tracks behind the live Sections. This is how they verify a section snaps to the lattice. Rules for picking spans

Author guidance — pick a span, don't invent one.

A short list of guardrails that keep Lingua pages legible across the catalogue. Most of these are about not doing things — the grid is most powerful when the variation across pages is small and the recipes are familiar.

1. Default to full-row. If you don't have a reason to split, leave the section at "12". A page of full-row sections reads like a paper; a page of mixed splits reads like a feed.
2. Favor balanced splits. 6 + 6 and 4 + 4 + 4 beat 5 + 7 and 3 + 5 + 4 unless the data justifies the asymmetry. Balanced splits read as composition; lopsided splits read as accident.
3. Use the integer spans for new work. "4" beats "1/3" because it reads directly against the 12-track ruler. The fractional aliases exist for back-compat; let them age out.
4. Row-spans only for hierarchy moments. A rowspan-2 next to two rowspan-1s is a hero pattern, not a default. Most pages should ship with all sections at rowspan: 1.
5. Don't override sectionPadding per-section. The grid's gap + the LayoutGroup's wrapper padding handle breathing room. Custom padding per-section is a code-smell — usually the layout calls for a different group variant instead.
6. Don't reinvent the grid per page. The same 12 columns serve marketing surfaces, dashboards, docs, and product chrome. If a page seems to "need" a different gridSize, that's almost always a sign the wrong LayoutGroup is being used.
7. Read the ruler before adding a section. Pick the span first against the ruler in §ruler, then drop the section in. Authors who write the span last end up nudging widths post-hoc; that's not how the grid wants to be used.
when these break down → They break down when a page is genuinely advanced — a dataset explorer with a sticky-left filter rail + scrolling body, or an editor with three resizable panes. Those cases use a layout-level grid (set on the Layout's childWrapper, documented in layouts.html), with the sectionArray grid still ruling each column. v0.2 doesn't ship this pattern yet; it's reserved for the page editor.

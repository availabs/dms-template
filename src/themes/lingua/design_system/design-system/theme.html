v0.2 design system · foundations

Theme — the tokens every surface inherits.

Color, type, icons, spacing, radii, elevation, motion, and the brand's signature dark code surface. The vocabulary Lingua speaks before it composes a sentence — everything in components and patterns reads from here.

Color

Colors — cool whites, electric blue, dark code surfaces.

~80% light surface + ink type. ~15% dark code/terminal surface (the brand's primary contrast move). ~5% one accent: Lingua blue. Status accents (signal/margin/success) are functional, not decorative. Pure white as the surface, near-black as the ink, electric ultramarine as the brand — no warm cream, no pastel tech-blue, no rainbow gradients.

color rules → The accent is one color (Lingua blue). A page emphasizing blue does not also emphasize signal or margin. Code surface is the brand's contrast move — appears at least once on every marketing page. Customer logos render in their native colors per the Warpstream convention. Single-hue gradients allowed (accent → accent-hover). Rainbows, frosted glass, and pure-black surfaces are not in vocabulary.
Type

Type — one sans, one mono, one editorial accent.

Inter Tight does 95% of the work, top to bottom. JetBrains Mono renders code, terminal output, KPI numerics, and the small mono labels that mark machine-readable surfaces. Fraunces Italic is the one editorial accent — reserved for the Theory page and the rare pull quote. The brand is sans-led; serif is a guest, not the host.

Token naming: {role}{Size}[{Variant}]. Roles: display · prose · mono · editorial. Sizes use the t-shirt scale Hero · XL · LG · MD · SM · XS. Body sets at 16px (developer-product standard, not the v0.1 publication 17). Color, italic, tabular-nums, uppercase are modifier axes, not new tokens.

Icons

Icons — geometric, line, 1.5px stroke.

Lucide or Phosphor (regular) as the working set; swappable for a custom drawn set without re-keying. Icons are functional — every icon is paired with a label or stands for a known action. Code-related icons (terminal, brackets, git, file-tree, copy, run, format) get extra attention because Lingua uses them often.

Spacing & radii

Spacing & radii — granular space, rounded by default.

4px base, 12-step scale. More granular than v0.1 because product surfaces need finer options. Corners are rounded by default — 4-6px on small chrome, 8px on cards, 12px on modals. Square corners are no longer the rule. v0.1's square-everywhere stance is retired.

Elevation & motion

Elevation & motion — soft shadows, snappy easing.

Three shadow tiers (sm/md/lg). Soft, neutral, never coloured. Cards lift on hover, modals float, popovers detach. v0.1's anti-shadow rule is retired — subtle depth is now part of the brand. Motion stays snappy: 120ms for hovers, 200ms for moves, 280ms for page transitions. No looping background animation.

--ease-soft · cubic-bezier(0.4, 0, 0.2, 1)
Activity feeds and terminal demos animate (typing effect, new row appearing). Background decoration does not. No parallax. No scroll-jacked drama.

Code surface

Code surface — the brand's signature contrast move.

Lingua's products and surfaces are typed rows; the typed shape is the brand. So every marketing page shows code, and code lives on a deep slate surface that contrasts with the light page. The shift between light surface and dark code is the brand's compositional fingerprint. Use it on at least one section per marketing page.

site/sections/issue-archive.row.js 
// every row, everywhere, has the same shape
export const issueArchive = {
  app:      "the-drift",
  type:     "issues|page",
  parent:   "the-drift:site",
  data: {
    title:     "Issue archive",
    layout:    "app",
    sections: [
      { component: "Card", source: "issues", columns: 6 },
    ],
  },
};

v0.2 design system · patterns · 13 compositions

Patterns — the moments where primitives meet.

Empty states, loading skeletons, filter bars, modal-with-form, the dataset list, the auth card — the recurring multi-primitive compositions every Lingua surface eventually renders. Plus two brand-signature additions for v0.2: the CLI command pattern and the architecture comparison — both Warpstream-inspired. Each section below is one composition; the chrome here is the chrome the live product ships.

01 · Empty state

No rows — the data section that hasn't been written yet.

A section whose source returned zero rows. Soft dashed border, sunken surface, mute icon, primary CTA in the brand voice ("Speak the first one"). Authors hit this every time they spin up a fresh dataset; the empty state is the brand's first impression.

No sections yet.

A page is a sentence; without sections, it has nothing to say. Pick a row format and Lingua will speak it for you.

Speak the first one → Browse templates
composition → state-card wrapper (sunken surface, dashed border) + Icon primitive (mute, large) + l-displayMD title + l-prose lede + Button primary + Button ghost. The CTA copy is brand voice ("Speak the first one"); the secondary is plain ("Browse templates"). Centered alignment marks this as a state, not a section body.
02 · Loading state

Loading — skeleton bars and a mono trace.

Skeleton blocks at the row level (not spinners) so the user sees the shape of what's coming. A small mono trace at the bottom shows what Lingua is doing — "Reading" on the brand's voice, with a pulsing accent dot.

Reading issues|page · 3 of 38 rows
composition → Surface card + 6 skeleton rows at varying widths (so the eye reads "different rows of different things") + a mono trace at the bottom. The trace tells the user what's actually loading ("issues|page", "3 of 38"). No spinners — spinners say nothing.
03 · Error state

Error — the diagnostic in a code block, then the retry.

Signal-colored left rule (4px), small signal icon, a title that says what didn't happen, the actual diagnostic in a dark .ling-code block, then a row of actions. The diagnostic is non-negotiable: a user who can read the trace can self-serve; a user who can't can copy it to a colleague.

04 · Data section

Data section with filters — the workhorse composition.

Header, filter bar inside --color-surface-soft, active filter chips, then the table, then pagination + page-size. This is the pattern Lingua renders more often than any other — every dataset list, every site index, every analytics view passes through this composition.

12 · CLI command v0.2 signature

CLI command — the terminal speaks the same row.

Lingua ships as both a GUI and a CLI. The two surfaces share the row format, so the CLI's output is just rows rendered to ANSI — same shape, different surface. This pattern is the brand's developer-product proof: "if you'd rather type, type."

~/sites/the-drift 
# scaffold a fresh site
lingua init the-drift --theme lingua
  resolving theme: lingua@0.2.4 · ok
  writing site:    the-drift:site
  writing pattern: the-drift|main:pattern
✓ initialized in 1.2s · cd ./the-drift

# add a page (a row is a row, whether GUI or CLI)
lingua add page issue-47 --layout default --section Card
  parent:  the-drift|main:pattern
  type:    main|page
  data:    { title: "Issue 47", slug: "issue-47", layout: "default" }
✓ added page · id=row_8c2a1f3e9d

# list every row of every kind for this site
lingua rows --site the-drift --kind page

the CLI shares the same row format as the GUI
composition → .ling-terminal primitive + window chrome (3 stoplight dots + cwd) + body of multi-line shell session. Prompt lines use --color-accent; output uses --color-code-mute for ambient text and --color-success for OK markers; row IDs and slugs get spot color to mark "this is the thing we just made." The session is real: init → add → rows → deploy walks the same lifecycle the GUI does, in 6 commands.

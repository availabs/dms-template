v0.2 two ways · one product

Hosted, or self-host. Same row format.

Lingua isn't two products. It's one open-source platform that you can either run on your own infrastructure or let us run for you. The vocabulary, the editor, the row format, the plugin API — identical. The difference is who owns the servers, and which trade-offs come with that choice.

One product

There's no upgrade path because they're the same product.

Most platforms with a hosted tier and an open-source tier treat one as the “real” product and the other as a feature-limited preview. Lingua is built the opposite way: the OSS repo and the hosted service share the same code, the same row format, and the same release pipeline. The hosted product is the OSS product with someone else running the servers.

That means there's nothing to migrate to. A site you author on hosted exports as a single archive that boots on a self-host install. A site you outgrew running on a 1 GB VPS imports into hosted the same way. Move when it suits you, move back when it doesn't, run both side by side — the choice belongs to the person paying the bills, not the vendor.

FAQ

Common questions, plainly answered.

The decision isn't permanent and the differences are smaller than they look. Here are the questions teams ask before they pick a door.

01Can I migrate from hosted to self-host?

Yes, both directions, no asterisks. lingua export produces a single compressed archive containing your pages, sections, datasets, themes, plugins, and history. lingua import reads it on the other end. The row format is versioned and the importer validates compatibility before writing anything — if a version mismatch would lose data, the import refuses and tells you what to upgrade.

02Does hosted run my custom plugins?

Public plugins from the marketplace: yes, one-click install on every tier. Private plugins of your own: yes on the Pro tier, with a review step before each new version is enabled — we read the code, because we share the runtime with every other hosted customer. Self-host has no such gate; if you trust the code, you can drop it into /plugins and restart.

03What's the real cost difference?

Hosted is $29 / site / month. Self-host is the cost of a small VPS ($5–$20 / month for most sites), a managed Postgres (free up to ~10 GB on several providers, ~$15–$50 / month after that), and your operator's time. Below 10 GB and one site, hosted is usually cheaper once you price the operator hours. Above 10 sites or with an in-house ops team, self-host wins quickly.

04Is the data format compatible across versions?

Yes. The row format follows semver. Minor and patch releases are fully backward-compatible. Major releases ship a one-shot migration in the importer; a v3 archive imports cleanly into a v4 instance. We have not yet shipped a major that broke compatibility for already-published sites and we don't plan to start.

05Who maintains the OSS version?

Lingua the company is the primary maintainer. The hosted product is built on the same repo — we don't keep a private fork — so the OSS gets the same attention the hosted product does, because they're the same code. External contributors merge ~30% of the commits in any given release; the maintainer team triages the rest.

06What about enterprise contracts?

Hosted Enterprise covers SSO/SAML, audit logging, custom data residency (US / EU / your-region), a signed BAA, and an SLA on uptime and support response. Self-host Enterprise is a paid support contract on top of the OSS — the software is still free; you're paying for an SLA-backed line to the maintainers. Either way, talk to sales: enterprise@lingua.com.

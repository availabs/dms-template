Structure · page chrome

Layouts — the chrome every page wears

Layout and LayoutGroup are the invisible structural layer: the hierarchy runs Site → Layout → LayoutGroup → Section → Component. A Layout owns the page’s chrome — TopNav, SideNav, the content column. A LayoutGroup is one band inside it. Nothing on this page is a visible component; everything on this page decides where visible components sit.

Hierarchy

Five layers, one nesting

Every page in this deliverable — and every page a DMS author composes — is this diagram. The annotation overlay you can see on this very page draws the same three inner layers in slate, oxide, and verdigris.

site — one row in the DMS; owns theme + patterns
layout `app` — outerWrapper · wrapper · wrapper2 · wrapper3
topnav — lives in wrapper2
sidenav — lives in wrapper3
childwrapper — the band stack
layoutgroup — wrapper1 (full-width band)
wrapper2 — mr-auto · max-w-content · pl-12 pr-8
section — one editable block on the 12-col grid · p-3 grout
component — card · graph · map · lexical · spreadsheet

The designer touches two of these layers on this page — Layout and LayoutGroup — and never sees them again: they render as class strings on wrapper divs. Sections and their components live on the grid page and the catalogue pages.

LayoutGroup variants

Eight bands, one recipe each

A LayoutGroup is two nested wrappers: wrapper1 is the full-width band, wrapper2 the capped container the sections sit in. Every variant the brand ships is catalogued here; a band that is not in this list is not a Tessera band.

content app · the default body band
wrapper1 w-full py-10 border-b border-grout-light
wrapper2 mr-auto w-full max-w-content pl-12 pr-8

The workhorse: hairline-divided courses of parchment cards, set against the SideNav. This band you are reading is one.

header app · page-title band
wrapper1 w-full border-b border-grout-light
wrapper2 mr-auto w-full max-w-content pl-12 pr-8 pt-12 pb-10

Unboxed: the title sits directly on bone. One per page, at the top; padding lives on wrapper2, not on a card.

band marketing · content band
wrapper1 w-full py-14
wrapper2 mx-auto w-full max-w-content px-6

The marketing counterpart of `content`: centred, because Layout `default` has no SideNav to hug. Generous vertical air, no hairline.

hero marketing · one per site
wrapper1 w-full pt-16
wrapper2 mx-auto w-full max-w-content px-6

The emblema’s frame: bone surface, no bottom padding on wrapper1 — the band closes with the tessera-course frieze instead.

band_limestone marketing · feature band
wrapper1 w-full bg-limestone border-y border-grout-light py-14
wrapper2 mx-auto w-full max-w-content px-6

The sunken feature band: full-bleed limestone with hairline rules top and bottom. At most one or two per marketing page — it is emphasis, not wallpaper.

auth bare or default · sign-in
wrapper1 w-full flex-1 flex p-6 items-center justify-center min-h-[80vh]
wrapper2 mx-auto w-full max-w-md bg-parchment border border-grout-light shadow-lift p-10

The one product surface that centres: a single parchment card held in the middle of the viewport. wrapper2 is the card.

footer marketing · page close

Full-bleed limestone closed by the meander frieze at its top edge — one of the frieze’s three sanctioned locations.

Wrapper reference

The class strings, verbatim

The variant cards convey shape; alignment decisions hide inside the class strings. This table is the contract the theme translation reads — one row per shipped variant, strings copied exactly.

Variant wrapper1 wrapper2 Alignment
content w-full py-10 border-b border-grout-light mr-auto w-full max-w-content pl-12 pr-8 left-set · 1280 cap
header w-full border-b border-grout-light mr-auto w-full max-w-content pl-12 pr-8 pt-12 pb-10 left-set · 1280 cap
band w-full py-14 mx-auto w-full max-w-content px-6 centred · 1280 cap (Layout default has no SideNav)
hero w-full pt-16 frieze-tiles at band base mx-auto w-full max-w-content px-6 centred · 1280 cap (Layout default has no SideNav)
band_limestone w-full bg-limestone border-y border-grout-light py-14 mx-auto w-full max-w-content px-6 centred · 1280 cap (Layout default has no SideNav)
auth w-full flex-1 flex p-6 items-center justify-center min-h-[80vh] mx-auto w-full max-w-md bg-parchment border border-grout-light shadow-lift p-10 centred (intentional)
footer w-full bg-limestone border-t border-grout-light meander frieze at top mx-auto w-full max-w-content px-6 py-12 centred · 1280 cap (Layout default has no SideNav)
workbench w-full py-6 w-full px-0 full-bleed · no cap

Product LayoutGroups on Layout app use mr-auto, not mx-auto. The codebase default (max-w-[1020px] mx-auto) drifts content off the SideNav’s right edge on wide monitors. The asymmetric pl-12 pr-8 gives the band breathing room from the SideNav without padding a right side that already borders open viewport. Marketing bands on Layout default centre because there is no SideNav to hug — an intentional, documented exception. Only auth centres on app.

Section width

Regular vs full-width groups

Inside any LayoutGroup, the sectionArray applies one of two layout classes to its grid container, picked by the group’s full_width flag. The literal code, from sectionArray.jsx:

className={`
  ${theme.container}
  ${theme.layouts[group.full_width === 'show' ? 'fullwidth' : 'centered']}
`}
group.full_width Class applied Result
'off' (default) layouts.centered = 'max-w-[1280px] mr-auto' sections capped at brand width, left-set against the SideNav
'show' layouts.fullwidth = '' sections fill the parent — no cap of their own

The toggle lives in the editor’s section-groups pane as a two-item menu (Full width: off / show) the author flips per group. Developers do not intervene.

Pairing rule. Setting full_width: 'show' on a content group still constrains sections, because that group’s wrapper2 caps at the brand width itself. True edge-to-edge needs the workbench LayoutGroup and full_width: 'show', so both layers drop their cap. Use it for map workbenches and wide tables only — the capped content band is correct for almost everything else.

Nesting

Bands stack into a page

A Layout holds a stack of LayoutGroups; composition is vertical. The marketing landing page reads, top to bottom, as four named bands:

hero
band
band_limestone
footer

One frieze per boundary, never stacked: the tessera course closes the hero, the meander opens the footer. Between them the floor stays bone, and the limestone band earns its contrast by being alone.

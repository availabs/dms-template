What's in the builder

The landing page gives the tour; this is the spec sheet. Every capability below ships in beta 0.6.0 and runs on production sites today.

Edit in place

The page is the editor. Click a section and its handles and toolbar appear around it — type, restyle, or rebind it without leaving the layout you're building.

  • What you edit is what visitors see — same layout, theme, and live data.
  • Changes save to a draft as you work; nothing goes live until you publish.
  • Undo exists, at every step.

draft → publish · autosave · undo

The sheet

Every page is sections on a 12-column grid. A section gets a column span and a row; the sheet handles alignment, wrapping, and responsive stacking on its own.

Drag to reorder, drag the edge to respan. Toggle the grid overlay and the page shows its own drafting lines — the same ones on this page's hero.

12 columns · spans 1–12 · drag to reorder · responsive by default

Data that stays live

Upload a file and it becomes a versioned dataset — columns typed, rows queryable. Cards, tables, charts, and maps bind to it directly.

Upload next month's file and every bound section updates. Old versions stay addressable, so last quarter's report keeps showing last quarter's numbers.

csv · xlsx · shapefile · geojson · versioned · queryable

the rest of the toolkit

Section types for real work

tables

Spreadsheet sections handle tens of thousands of rows: column types, formatted numbers, links, pills. A keyword search over the whole table is one toggle.

sort · filter · paginate · search

charts

Pick a dataset, an x, and a y. Charts render in the site's palette, and a target line is just another series — no custom code on either side.

bar · line · area · target lines

maps

GIS uploads render as real vector maps — parcels, corridors, county lines — with popups bound to the same dataset as everything else.

vector tiles · parcels · corridors · popups

themes

Every button, nav, and card reads from theme tokens. Swap the theme and the same content becomes a different site — dark mode included.

tokens · named styles · light + dark

drafts & publish

Published pages are immutable snapshots. Visitors never see drafts, an editor bug can't break a live page, and rolling back is picking a version.

drafts · snapshots · rollback

access

Sign-in and roles are built in. Public pages stay public; edit rights go to the people who own the content — DJs edit their show pages, staff edit listings.

sign-in · roles · per-page access

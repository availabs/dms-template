Components

Every primitive an author can drop on a page, drawn in this brand's hand: nav in both directions, every form control in every state, overlays as still photographs, the dataCard workhorse with its knobs labelled, and the rich sections — text, table, graph, map. If a live render doesn't match this page, the theme is wrong, not the page.

the demo data is real — bird-log and traffic_counts exist on my beta ↓

01 · navigation

TopNav, SideNav, menus, crumbs

Nav is hairlines and one accent. The active item is the only cobalt thing in the chrome; everything else is graphite that darkens to ink on hover. No fills, no shadows — a border-b and a soft cobalt wash do all the work.

03 · overlays

The one place shadows are allowed

Overlays float above the sheet, so they get shadow-drag — the only elevation deeper than a 1px lift. Backdrop is the scrim token. These are static snapshots; the live ones animate open in 150ms and trap focus.

04 · containers & atoms

The dataCard is the product

Authors don't write components — they configure Cards. A Card binds to a dataset, picks columns as cells, and lays them on the same 12-column sheet as everything else. Every knob shown here (cardsGridSize, cellSpan, valueFontStyle…) is a control in the section toolbar, not a line of code.

Card — the generic container

anatomy

bg-panel · border-rule · rounded-lg · p-6

05 · rich content

Text, tables, graphs, maps

The four section types that carry actual content. Lexical inherits the display ladder wholesale; tables and graphs put every numeral in tabular figures; the map is a sheet with drawings on it, which is the most tessera sentence on this page.

Lexical — rich text

lexical.* — headings map to display tokens

Keeping a bird log h1 → displayLG

What counts as a sighting h2 → displayMD

Porch rules h3 → displaySM

A sighting is a bird I can identify with confidence, seen from the porch, while the coffee is still warm. Everything else goes in the maybe list for the weekend.

  • binoculars stay on the rail, always
  • photos are optional; the log entry is not
  • ties go to the more boring species

"the data model for a hobby should be smaller than the hobby."

Counts sync to the dataset as bird_sightings.count the moment the row saves.

// bind a section to data

section.source = "bird_sightings"

section.columns = ["species", "count", "seen_at"]

publish("bird-log") // that's the whole job

code blocks live on t6-board: chalk base, go strings, amber props, chalkDim comments. cobalt stays off the board — it's too dark there.

The basemap is the sheet itself — t6-sheet with roads in ruleStrong. Data layers use soft fills with their solid token as the stroke, so the map reads like a marked-up drawing, not a satellite photo. Controls and legend are ordinary panels: border, rounded-md, shadow-lift.

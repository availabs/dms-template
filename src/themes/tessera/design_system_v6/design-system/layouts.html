tessera design system · v6 · graph paper

Layouts — page chrome

Layout and LayoutGroup are the invisible chrome: the Layout wraps the whole page (nav or no nav), and LayoutGroups are the horizontal bands your sections stack into. Authors never see either one directly — they just pick a name, and these class strings do the rest.

you're inside a Layout right now — sidenav, topnav, this band. that's the whole trick ↖

01 · hierarchy

Five layers, one nesting

Every DMS page is the same russian doll: a Site serves a Layout, the Layout stacks LayoutGroups, each group grids out Sections, and each section renders one Component. This page documents the two outer shells; grid.html covers how sections sit inside a group.

site

routes, auth, data — everything below happens per page

layout

the chrome — topnav, sidenav, and the content column

layoutgroup

one full-width band; a page stacks several

section

component — card, text, graph, map…

section

component

The two inner layers are where authors live — they drag sections and configure components. The two outer layers are where the brand lives: this page pins down exactly what each named Layout and LayoutGroup renders, so authors get consistent chrome for free.

02 · layout variants

Three chromes: default, app, bare

A Layout is five nested wrappers (outerWrapper → wrapper → wrapper2 → wrapper3 → childWrapper) plus optional TopNav and SideNav. v6 ships three named recipes — pick by the kind of page, not by taste.

default

marketing

TopNav only, generous air, content centred — there's no sidenav to hug, so landing pages sit in the middle of the sheet.

outerWrapper — paper fill, min-h-screen; the sheet itself

wrapper2 — carries the TopNav + content column; bands centre with mx-auto (the documented exception, §04)

childWrapper — LayoutGroups stack edge-to-edge, full width

app

product · this page

SideNav + slim TopNav. Content hugs the sidenav's right edge — bands use mr-auto, never mx-auto.

wrapper3 — the flex row: SideNav | content; owns the hairline between them

childWrapper — content column, min-w-0 so wide tables scroll instead of pushing the nav

wrapper — min-h-screen flex; collapses the SideNav below lg

bare

auth · embeds

No navs, no chrome at all. The sheet fills the viewport and one group (usually auth) centres its content in it.

outerWrapper — paper fill, min-h-screen; the only wrapper doing work

wrapper · wrapper2 · wrapper3 — pass-throughs; both navs set to size: 'none'

childWrapper — flex flex-col min-h-screen so the auth group can centre vertically

03 · layoutgroup variants

Six bands, one recipe each

Every band on every v6 page is one of these six. The surfaces are deliberately quiet — a hairline under each band, the sheet showing through in the gutters — so the loud ones (feature, board) read as events, not wallpaper.

header

unboxed · border-b

The title band. Text sits directly on the sheet — no panel, no box. One per page, always first.

content

the default

The workhorse band. Sections bring their own panel chrome; the band itself is just sheet + a bottom hairline.

feature

marketing only

The highlight band: 16px hatched strips at the viewport edges and hatched t6-divider strips separating it from its neighbours, square joint tiles pinned where the rules meet. Every marketing band picks one edge pattern (hatch · backhatch · dashes · ticks · rungs) so the dividers stay connected — the catalog lives on theme.html §the sheet.

board

full-bleed dark

The dark band — code, terminal art, the "here's the actual product" moment. Chalk text on t6-board; it keeps its own tokens in both lamps.

terminal

npx tessera new my-site

footer

quiet close

A slim band, no bottom hairline (nothing follows it), py-8 instead of 12. Link rows and the colophon live here.

auth

centred · intentional

The one band that centres: place-content-center, capped at max-w-md. Lives on Layout bare, fills the viewport.

04 · wrapper classes

The class strings, verbatim

The variant cards above show shape; this table is the source of truth. The mr-auto / mx-auto decision hides inside these strings — copy them exactly, because alignment drift is invisible until someone puts two pages side by side.

variant wrapper1 wrapper2 alignment
header w-full border-b border-rule mr-auto w-full max-w-content pl-6 lg:pl-12 pr-6 lg:pr-8 py-12 left-aligned · cap
content w-full border-b border-rule mr-auto w-full max-w-content pl-6 lg:pl-12 pr-6 lg:pr-8 py-12 left-aligned · cap
feature w-full border-b border-rule relative mr-auto w-full max-w-content pl-6 lg:pl-12 pr-6 lg:pr-8 py-16 left-aligned · cap
board w-full t6-board mr-auto w-full max-w-content pl-6 lg:pl-12 pr-6 lg:pr-8 py-16 left-aligned · cap
footer w-full mr-auto w-full max-w-content pl-6 lg:pl-12 pr-6 lg:pr-8 py-8 left-aligned · cap
auth w-full flex-1 flex p-6 mx-auto w-full max-w-md flex flex-col place-content-center centred (intentional)
marketinglayout default only w-full border-b border-rule mx-auto w-full max-w-content px-6 lg:px-8 py-16 centred (documented exception)

Product LayoutGroups on Layout app use mr-auto, never mx-auto. The codebase default (max-w-[1020px] mx-auto) centres bands on wide monitors, so content drifts away from the sidenav's right edge and the page reads marketing-y. mr-auto keeps every band hugging the sidenav, with pl-6 lg:pl-12 for breathing room. Only two surfaces centre on purpose: auth forms, and marketing bands on Layout default — which has no sidenav to hug.

05 · section width

Regular vs full-width groups

Inside any group, the section grid picks one of two layout classes based on the author-facing Full Width: off / show toggle. This is the literal line from sectionArray.jsx — no developer in the loop.

sectionarray.jsx · the whole mechanism

className={`
  ${theme.container}
  ${theme.layouts[group.full_width === 'show'
    ? 'fullwidth' : 'centered']}
`}

container = "w-full grid grid-cols-12 gap-0"

full_width class applied result
'off'default layouts.centered =
'max-w-content mr-auto'		 sections capped at 1200px, hugging the sidenav
'show' layouts.fullwidth = '' sections fill the parent — maps, dense tables, chart canvases

Heads up: 'show' only removes the inner cap — the group's wrapper2 still caps at max-w-content. For true edge-to-edge (a map workbench), pair it with a workbench-style group whose wrapper2 drops the cap entirely.

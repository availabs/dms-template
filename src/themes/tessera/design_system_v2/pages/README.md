# pages/

Optional example page(s) demonstrating what the Tessera theme is *for*.
Per `references/dms product/design-system-for-dms.md` §12.6, this folder
is **theme-chosen and optional** — themes are choices, and a theme
designed for a particular use case doesn't have to validate against
every kind of surface the platform supports.

## What ships in v0.1

| File | Demonstrates |
|---|---|
| `civic-overview.html` | A county-level hazard mitigation overview (Sullivan County, NY). KPI strip, hazard map with NFIP claims overlay, stacked-bar status chart, recent-actions table, stakeholder list. The surface Tessera is most clearly designed for — multi-stakeholder, durable, civic. |

## Why this one example

The Tessera brand register is **architectural, considered, durable,
quiet** — closer to a civic press or typeface foundry than a tech
startup. The most natural target is the civic/government data surface:
hazard mitigation plans, public dashboards, environmental reports,
infrastructure overviews. The MitigateNY-shaped example is the strongest
single proof that the brand can render that kind of work with
seriousness.

## What's deliberately not here

- A wcdb-shaped radio-station home page. Possible, but the brand's
  voice and palette aren't a natural fit for that aesthetic — Tessera
  reads "civic ordinance," not "weekend mixtape."
- An NPMRDS-shaped federal analytics page. Could work; the chart
  treatment and tabular figures are right for it; but one civic example
  is enough to make the point for v0.1.
- An AVAIL-docs-shaped documentation site. The four design-system
  pages already function as technical-doc surfaces; another one would
  be duplicative.

A future pass should add an example page only when there's a clearly
distinct use case the brand wants to claim.

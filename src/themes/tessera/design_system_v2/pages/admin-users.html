DESIGN NOTES · GAPS VS. CURRENT

What this design adds over the current implementation.

Added
  • + KPI strip: total, active-30d, pending, group count
  • + Tab strip: All / Pending invites / Access requests
  • + Avatar with initials (colour-keyed to user index)
  • + Group pills colour-coded: oxide=admin, verdigris=editor, neutral=viewer
  • + Status dot: active (verdigris) / inactive (fog) / pending-invite (ochre)
  • + Pending row: striped bg + invitation date + Resend / Cancel actions
  • + View As icon button (filled=active) vs. secondary button in current code
  • + Sidenav drill-down by group
Implementation notes
  • → Active status: last_login within 30 days
  • → Pending: user.state === 'invited' or no last_login + created within 7d
  • → Group pill colours: check if group name contains "Admin" → oxide; "Edit" → verdigris; else neutral
  • → Resend: call /signup/assign/group with same email again
  • → Sort columns: client-side sort of the loaded users array

The grid.

Every section on every Tessera page sits on one 12-column grid — the sectionArray — inside whatever band hosts it. This is where the terminal and the mosaic are the same object: a fixed matrix of cells, composed in courses. There are no freehand widths anywhere in this deliverable.

The grout model

gap-0 · padding gutters · inner-box chrome

The grid itself is gap-0. The gutter is p-3 padding on every section, and a section's chrome — border, background — renders on an inner box inside that padding. In mosaic terms: cells are set edge to edge; the grout is part of each tessera's bed, not a property of the wall. This is what lets adjacent sections either sit apart or fuse into one visible card by zeroing the shared edge.

Anatomy — one section

section wrapper · col-span-N + p-3 ← the grout

inner box · border + bg ← the chrome

component content — its own padding is the component's concern, never conflated with the gutter.

Picking spans

  • Default to 12. A full course is never wrong.
  • Favor even courses — 6+6, 4+4+4, 3×4 — over asymmetric splits unless the content earns the emphasis (then 8+4).
  • KPI rows are 3s; card grids are 4s; prose next to its aside is 8+4.
  • Never sub-3 — a tessera smaller than a quarter course can't hold chrome and content.
  • Row-spans are punctuation, not rhythm.

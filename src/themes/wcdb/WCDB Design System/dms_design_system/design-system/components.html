Foundations · components · v0.1

Primitives.

Every UI primitive shipped by @availabs/dms, skinned in the WCDB voice. Italic display headlines for hierarchy, mono uppercase eyebrows for chrome, Geist sans for body — and a single functional red reserved for live-broadcast signals.

Each block below references the theme key that produces it. Page authors do not reach into this gallery — they pick from these surfaces via the section toolbar. See src/themes/CLAUDE.md for the rule: configure the Card, don't write a new component.

04 — Containers & atoms

Card & chrome.

The Card section is the workhorse of every page. Pills, icons, tables, the logo — small things, but the alphabet that page authors compose with.

Card · card.theme.jsx · dataCard

The workhorse.

Author empowerment

Before reaching for a custom column type or section component, see src/themes/CLAUDE.md — configure the Card, don't write a new component. The Card grid expresses image, text, formatted-number, link, and span layouts. A custom column type is the right tool only when there is no underlying value to render (play button, on-air chrome).

06 — WCDB column types

Theme-registered cells.

Three small column types — each renders one visual element so the Card grid can compose with them alongside ordinary data cells. Registered via theme.columnTypes; auto-picked up by patterns/page/siteConfig.jsx.

portrait_banner · portraitBanner.config.js

DJ banners.

fullBleed: true, spanFullColumns: true, defaultHideHeader: true. The value is hashed into a hue; initials overlay through the .wcdb-initials utility. Height is picked from theme.portraitBanner.bannerHeights.

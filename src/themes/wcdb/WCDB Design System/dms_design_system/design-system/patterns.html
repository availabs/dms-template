Compositions · patterns · v0.1

Patterns.

The multi-primitive compositions that recur across WCDB. Each pattern below is a Section (or a small chain of Sections) authored by configuring Card cells — never a custom component. The first block documents the standard DMS patterns; the second transcribes the brand-specific compositions from the handoff.

01 — Empty state

Quiet on the air.

Data sections with no rows render an icon, an editorial italic-display message, and a single call-to-action. Kept monochrome — never the on-air red.

No spins yet.

Tonight's playlist hasn't started logging. The DJ goes live at 11 pm — check back then or browse last week's spins.

02 — Loading state

Tuning the dial.

A Card outline with grey-shimmer row placeholders. Alternating --bg-2 / --bg-3 so the rhythm reads even before content arrives.

03 — Error state

Dead air, briefly.

Errors stay monochrome — the on-air red is reserved for live broadcast signals. Diagnostic detail rides in metaXS under the message.

Couldn't reach the studio.

The spins feed didn't respond. Try again in a moment — we're usually back within thirty seconds.

Err · WCDB-504 · upstream timeout · 18:42 EST

Last refresh, 14 min ago.

A soft hairline banner above stale tabular data. The metadata rides in the mono eyebrow voice; refresh sits flush right as a quiet pill button.

§ 7.6.5 — Signature patterns

WCDB signature patterns.

Brand-specific compositions transcribed verbatim from design_handoff_wcdb_design_system/README.md. These appear on multiple pages and define how WCDB feels distinct from a generic editorial theme.

A — Bare-vs-carded mix

The right column breathes.

Most design systems put every section in a card. WCDB deliberately alternates bare sections (on the page bg, the typography carries them) with carded sections (functional content earns a panel). This is the brand's most distinctive composition rule.
